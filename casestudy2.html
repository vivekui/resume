Product Features
1. API Specification (XML Based)

Our team takes care of all developer related items around APIs, that include API Design, API Automation, SDK Generation, Help Doc writing. Our work has to be streamlined and optimized, so that we don't do any repeated work and reuse the already taken design decisions. We designed our own Specification to represent APIs, based on the concepts taken from OpenAPI.

2. Automated Automation

In Zoho CRM, the API functionalities are huge. Hence is our total API usecases. With the advent of API Spec, we write the automation code only for the specific usecases, while others are being covered by default.

3. SDK & Help Doc Generation

Coding in each and every language for same kind of API usecase is mundane. It is alsp prone to manual errors. So it is all automated.

I am not mentioning other features due to the disclaimer.
Challenges & Solutions
1. Faster Automation Time

In Zoho CRM, the no of API Automation usecases are large. So firing those API calls and checking the response sequentially is time consuming. So we used ApacheNIO for asynchronous API firing and processing the response. Also, we added more servers to serve those large no of API calls. Thus saving a huge amount of time.

2. Running Automation Online

Automated automation can be run locally. But what if the developer's machine configuration is not good enough and they want to run their automation elsewhere? We developed an architecture where multiple automations can be run parallely with proper system resource allocation. Not enclosing other details.

3. Faster API Audit

When doing the API Design, it is mandatory for us to follow standards across the product and check those each time during the Audit phase of the API Design. Since we follow the specification, the tool can do all the basic checks by default and the auditor can work only on the usecases which are specific to the particular feature.
